## 二、治理架构设计

### 2.1 目标

用 **DoDAF 2.0 (DM2)** 框架治理知识库本身，实现：
1. **结构化**：每个知识对象都有明确的 DM2 定位
2. **可追溯**：每个笔记都有来源、版本、责任人的谱系记录
3. **可度量**：知识覆盖率、质量、时效性可量化评估
4. **自演进**：治理方案本身也在框架内迭代优化

### 2.2 DM2 数据组 ↔ 知识库映射

| DM2 数据组              | 知识库对象              | 示例                             |
| -------------------- | ------------------ | ------------------------------ |
| **01-Performer**     | 知识角色（人/AI/系统）      | 雷（owner）、Claw（AI）、Obsidian（工具） |
| **02-Activity**      | 知识生命周期活动           | 创建→审核→发布→归档→销毁                 |
| **03-Capability**    | 知识管理能力             | 检索、关联、推荐、备份、版本控制               |
| **04-Resource**      | 知识资产               | 笔记、附件、标签、元数据                   |
| **05-Guidance**      | 治理规范               | 分类标准、命名规范、Frontmatter 模板       |
| **06-Measure**       | 治理度量               | 知识数量、覆盖率、重度率、时效性               |
| **07-Location**      | 存储位置               | 五大分类目录、子分类                     |
| **08-Services**      | 工具服务               | Obsidian 插件、AI 能力              |
| **09-Project**       | 治理项目               | DM2 知识库完善、Inbox 清理             |
| **10-Reification**   | Type/Individual 分层 | 模板（Type）→ 实例（Individual）       |
| **11-Temporal**      | 时间维度               | 创建时间、修改时间、有效期                  |
| **12-Naming**        | 命名规范               | `YYYY-MM-DD-标题.md`             |
| **13-Description**   | 描述模板               | Frontmatter schema             |
| **14-Pedigree**      | 血缘追踪               | 作者、版本、来源、审批状态                  |
| **15-Org Structure** | 组织结构               | 五大分类 + MOC 导航                  |
| **16-Rules**         | 业务规则               | "新笔记48小时内必须整理"                 |
| **17-Associations**  | 关联关系               | 双链 [[链接]]                      |
| **18-Views**         | 治理视图               | AV-1、OV 系列、SV 系列               |
|                      |                    |                                |

### 3.5 G-StdV-1：治理标准规范

| 标准类型     | 规范内容                                           | 约束级别 |
| -------- | ---------------------------------------------- | ---- |
| **N-01** | 新笔记必须先放入 `0Inbox/`                             | 强制   |
| **N-02** | Inbox 内容 48 小时内必须整理                            | 强制   |
| **N-03** | 每个笔记必须填写 `title`、`date`、`category` frontmatter | 强制   |
| **N-04** | DM2 相关笔记必须填写 `dm2-type`、`dm2-dataGroup`        | 强制   |
| **N-05** | 笔记之间必须使用 `[[双链]]` 关联                           | 推荐   |
| **N-06** | 文件名格式：`YYYY-MM-DD-标题.md`                       | 推荐   |
| **N-07** | 定期（建议每周）进行 Inbox 整理                            | 强制   |
| **N-08** | 归档笔记移动到 `文学/归档/`                               | 推荐   |
