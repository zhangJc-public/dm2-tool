3. **SV/SvcV 镜像轨**之间未文档化的 1:1 对应关系

## 二、研究材料分析

### 2.1 DM2 Data Dictionary — Monster Matrix 编码

DM2 Excel 的 Sheet "Data Dic, Aliases, Mappings" 包含 514 行 DM2 概念，每行映射到 52 个 DoDAF 视图列 (cols 28-79)，使用以下编码：

| 标记 | 含义 | 推断 |
|------|------|------|
| `n` | 该概念被此视图**实例化/使用** (needed) | 核心数据依赖 |
| `o` | 该概念**类型**被此视图引用 (optional reference) | 元数据引用 |
| 空 | 该概念与此视图无关 | — |

**DM2 概念在各视点的分布**：

| 概念域 | n-views (定义此概念的视图) | o-views (引用此概念类型的视图) |
|--------|--------------------------|------------------------------|
| Capability | CV-1~7, PV-3 (8 views) | 10 views |
| Activity | 36 views | 49 views |
| System | SV-1~11, SvcV-3a (12 views) | 37 views |
| Service | SvcV-1~10c, CV-7 (14 views) | 26 views |
| Performer | 18 views | 48 views |
| Resource | 29 views | 50 views |
| Information | DIV-1/2/3, OV-5b (4 views) | 24 views |
| Data | DIV-2/3, SV-4, SV-6, SvcV-3b, SvcV-6 (6 views) | 15 views |
| Project | AV-1, CV-3, CV-5, PV-1~3 (6 views) | 7 views |
| Standard | SV-2/6, SvcV-2/6, StdV-1/2 (6 views) | 32 views |
| Rule | OV-3, StdV-1/2 (3 views) | 44 views |
| Measure | 21 views | 47 views |

**关键洞察**：
- **Activity 是 DoDAF 中最普遍的概念** — 36 views 将其作为核心数据，49 views 引用其类型。Activity 是整个框架的"通用语言"。
- **Rule 仅在 3 个视图中实例化** (OV-3, StdV-1, StdV-2)，但被 44 个视图引用 — 规则是使用最广的元数据。
- **Information 和 Data 之间的边界清晰**：Information 在 OV-5b → DIV-1 链中产生，Data 从 DIV-2 开始出现。

### 2.2 DoDAF 2.02 规范中的关系描述模式

从 PDF 中提取的关系描述遵循以下模式：

| 模式 | 示例 | 关系类型 |
|------|------|---------|
| "X 是 Y 的对应模型" | SV-4 是 OV-5b 的系统视点对应模型 | **镜像** (SV↔OV) |
| "X 由 Y 实现" | SV-1 实现 OV-2 的逻辑架构 | **实现** (logical→physical) |
| "X 来源于 Y" | SV-8 中描绘的变化来源于 PV-2 | **数据依赖** |
| "X 需要与 Y 关联" | StdV-2 需要与 SV-8/SvcV-8/SV-9/SvcV-9 关联 | **同步依赖** |
| "X 基于 Y" | SV-7 基于 SV-6 构建 | **构建依赖** |

## 四、SV/SvcV 镜像对称性验证

### 4.1 完全对称的视图对

DM2 Excel 和 DoDAF 规范均确认以下 1:1 对应：

```
作战层源    系统轨(SV)        服务轨(SvcV)        差异
────────    ────────         ──────────         ────
OV-2    →   SV-1  接口   ↔  SvcV-1  接口      点对点 vs 发布/订阅
OV-2    →   SV-2  资源流 ↔  SvcV-2  资源流    协议约束相同
OV-5b   →   SV-4  功能   ↔  SvcV-4  功能      功能粒度不同
OV-5b   →   SV-5a 追溯   ↔  SvcV-5  追溯      —
OV-3    →   SV-6  矩阵   ↔  SvcV-6  矩阵      —
—       →   SV-7  度量   ↔  SvcV-7  度量      SLA 侧重不同
OV-2    →   SV-8  演进   ↔  SvcV-8  演进      SOA 转型 vs 系统升级
—       →   SV-9  技术   ↔  SvcV-9  技术      —
OV-6a   →   SV-10a 规则  ↔  SvcV-10a 规则     编排约束(SvcV独有)
OV-6b   →   SV-10b 状态  ↔  SvcV-10b 状态     —
OV-6c   →   SV-10c 时序  ↔  SvcV-10c 时序     —
```

### 4.2 不对称部分

| 视图 | SV特有 | SvcV特有 | 原因 |
|------|--------|---------|------|
| SV-3 | 系统-系统矩阵 | — | 对应 SvcV-3b |
| SvcV-3a | — | 系统-服务矩阵 | 跨域映射 |
| SvcV-3b | — | 服务-服务矩阵 | 对应 SV-3 |
| SV-5b | 活动→系统追踪 | — | 对应 SvcV-4 的能力映射部分 |

**结论**: 10/11 对完全对称。唯一的不对称来自：
- SV-3 ↔ SvcV-3b (1:1 对应但编号不同)
- SvcV-3a 是额外的跨域桥梁 (系统↔服务)
- SV-5b 是 SV-5a 的扩展 (双向矩阵)
