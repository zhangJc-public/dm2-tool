# DoDAF 视图生成协议 (View Generation Protocol)

> **用途**: 每次生成任何 DoDAF 视图（OV-1/SV-4/CV-2 等）之前，**必须先执行本检查清单**。
>
> **触发条件**: 用户要求输出任何以视图编号命名的文档（如"画一个 OV-1"、"写一份 CV-1"）。
>
> **创建原因**: 2026-04-19 实际事故——生成 OV-1 时默认了 CV-1→OV-1 = 1:1，但 P0-OV-1 报告中早有 1:N 的明确记录。根因是**生成前未检索已有知识 + 线性默认值陷阱**。

---

## 第一阶段：预检（Pre-flight Check）

### 1.1 知识检索（必须执行）

在写任何代码/文字之前，按以下顺序读取已有文档：

```
① 视图定义 → MCP: get_view_definition(view_id)
② 输出模板 → MCP: get_output_template(view_id)
③ 已有分析报告 → search_file("P0-{view_id}*" 或 "P1-{view_id}*")
④ METHOD 方法论 → read_file("METHOD-MethodologyAnalysis.md") 相关章节
```

> ❌ **禁止跳过**: 2026-04-19 的事故就是因为没做第③步。

### 1.2 关系约束检查（强制）

以下关系规则来自知识库已有分析，**不是可选建议而是硬性约束**：

| 规则编号 | 关系 | 强制内容 | 来源 |
|---------|------|---------|------|
| **R1** | **CV-1 : OV-1 = 1 : n** | 一个愿景必须拆解为 ≥1 个场景概念图。拆分维度：时间轴(As-Is/To-Be) / 业务域 / 关注方 | P0-OV-1 §1.2 + §3.1; METHOD §发现4 |
| **R2** | **AV-1 : OV-1 = 1 : n** | 一个执行摘要对应多张场景图 | P0-OV-1 §1.2.3 |
| **R3** | **OV-5b → SV-4 → SvcV-4** (三层递进) | 同一活动在业务/系统/服务三层逐级细化，不可跳层 | METHOD §4.3.3 |
| **R4** | **OV-6a → SV-10a → SvcV-10a** (规则三层) | 业务规则/技术系统规则/服务契约规则逐级具体化 | METHOD §4.3.6 |
| **R5** | **StdV 是 Why 的法理终点** | 回答"为什么"时必须同时列出遵循的标准清单 | METHOD 发现2 |

### 1.3 CV-1→OV-1 专用检查清单

当生成 OV-1 时，**逐条回答以下问题并写入文档**：

| # | 问题 | 如果"是"则动作 |
|---|------|--------------|
| C1 | 项目是否有明确的 As-Is 和 To-Be 状态差异？ | 至少产出 2 张 OV-1（基线 + 目标） |
| C2 | 是否涉及 ≥2 个独立的业务域或任务场景？ | 按域各产出 1 张 OV-1 |
| C3 | 是否有安全/合规视角需要独立描述？ | 补充安全域 OV-1 |
| C4 | 不同干系人的信息需求能否用一张图满足？ | 按干系人拆分 OV-1 |

> 如果 C1~C4 全部为"否"，才允许产出单张 OV-1。**默认假设应该是需要多张，而非一张。**

## 第二阶段：生成（Generation）

### 2.1 文档结构要求

每份视图文档必须包含：

```markdown
## 一、定位与目的
   - 回答什么 6W 问题？
   - 与 CV-1 的关系（哪个愿景派生？）
   - 本 OV-1 在 CV-1:OV-1 = 1:n 中的位置（第几张？聚焦什么？）
## 二、核心内容
   ...
## 三、关联视图
   - 上游（哪些视图的输入？）
   - 下游（驱动哪些视图？）
   - 同级（其他 OV-1 子视图？）
```

### 2.2 多 OV-1 场景的命名规范

当存在多张 OV-1 时：

```
OV-1a: [项目名] - 基线架构 (As-Is)
OV-1b: [项目名] - 目标架构 (To-Be)        ⭐ 通常主文档
OV-1c: [项目名] - [业务域A] 场景
OV-1d: [项目名] - [业务域B] 场景
OV-1e: [项目名] - 安全应急场景
```

## 第三阶段：自检（Post-generation Review）

产出后逐项确认：

| # | 检查项 | 通过标准 |
|---|--------|---------|
| V1 | CV-1 在本文档之前被引用或内联了吗？ | ✅ CV-1 必须出现在 OV-1 之前或同文档前部 |
| V2 | 如果是多 OV-1 项目，其他子视图被标注了吗？ | ✅ 至少列出"本项目的完整 OV-1 清单" |
| V3 | 视图路由表中的 OV-1 条目体现了 1:n 吗？ | ✅ 不应只有一行"OV-1"，应有拆分行 |
| V4 | P0/P1 分析报告中的已知模式被复用了吗？ | ✅ 不应重复发明已有的分析方法 |
