# PV-1: Project Portfolio（项目组合）

## 一、定位与目的

### 1.1 这个表回答什么问题？

**一句话**：为了交付这些能力，需要哪些项目/采办项目？

PV-1 是项目视点的**顶层总览**——将能力需求转化为具体的项目投资组合。

核心问题集：
| 问题 | 回答 |
|------|------|
| 有哪些项目在推进这个架构？ | 项目清单与分类 |
| 每个项目支持什么能力？ | 能力→项目映射 |
| 项目之间如何关联？ | 依赖和协同关系 |
| 总体投资规模如何？ | 资源分配概览 |

## 二、核心内容解析

### 2.1 必要性说明

> PV-1 描述**支持架构描述所需的项目或采办项目的集合**。

PV 视点的核心价值是将**能力需求（CV）转化为可执行的项目投资（PV）**。PV-1 是这一转化的入口。

### 2.2 内容要素

| 要素 | 说明 |
|------|------|
| **项目名称与标识** | 唯一标识每个项目 |
| **项目目标** | 该项目要达成的能力效果 |
| **能力支撑关系** | 项目→能力的映射（多对多） |
| **项目间依赖** | 前置/并行/后续关系 |
| **资源估算** | 粗粒度的成本/人力/时间 |
| **状态** | 规划中 / 进行中 / 已完成 |

### 2.3 与其他视图的关系链

```
CV-2(能力) + CV-3(时间线)
        ↓ 驱动
   PV-1(项目组合) ← 本视图
        ↓ 细化
   PV-2(时间线) — 每个项目的详细排期
        ↓ 映射
   PV-3(项目-能力) — 双向追溯验证
```
