# CV-3: Capability Phasing（能力阶段规划）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：这些能力**什么时候**可用？有没有时间缺口？

CV-3 是能力的**时间维度投影**——将 CV-2 的静态层级展开到时间轴上，展示每个能力在不同阶段的交付状态。

核心问题集：
| 问题 | 回答 |
|------|------|
| 能力A什么时候上线？ | 具体阶段/时间段 |
| 这个阶段缺什么能力？ | 空白 = 差距 |
| 有没有重复建设的能力？ | 重叠 = 冗余 |
| 哪些项目支撑这个能力增量？ | 项目→能力→时间的关联 |

### 1.2 在架构描述中的角色

```
┌─────────────────────────────────────────────────┐
│              CV-3: 能力阶段规划                   │
│         （能力的时间维度 / 交付路线图）            │
├─────────────────────────────────────────────────┤
│                                                  │
│   能力 (来自CV-2)  │ 现状 │ 过渡期 │ 目标 │ 未来  │
│   ────────────────┼──────┼───────┼──────┼─────  │
│   态势感知        │ ████ │ █████ │ █████│      │
│   指挥控制        │ ██░░ │ ████░ │ █████│      │
│   网络防御        │ ░░░░ │ ██░░░ │ ████ │ ████ │
│   后勤保障        │ ████ │ ████  │ ████ │      │
│                                                  │
│   ████ = 已具备    ░░░ = 缺口                    │
│   长色条 = 跨多阶段的持续能力                     │
│                                                  │
└─────────────────────────────────────────────────┘
```

**关键角色**：
- **能力审计工具**——识别差距（gap）和冗余（duplication）
- **采购决策支撑**——将能力增量与项目里程碑关联
- **投资组合同步器**——帮助 Portfolio Manager 同步跨项目的能力引入
- **CV-2 的动态视图**——从"有什么能力"到"什么时候有"

### 与其他 CV 视图的协作关系

```
CV-1 (愿景) ──定义阶段──→ CV-3 (列: 阶段)
                              ↑
CV-2 (分类法) ──提供行──→ CV-3 (行: 能力)

CV-3 的产出:
  ├──→ CV-4 (能力依赖: 时间上的依赖关系)
  ├──→ PV-2 (项目时间线: 能力交付的数据来源之一)
  └──→ 差距分析报告 (Gap Analysis Report)
```

## 二、核心内容解析

### 2.1 必要性说明（官方 Purpose 翻译）

> CV-3 解决在**不同时间点或特定时间段内**计划实现的能力的问题，即**能力阶段规划**。CV-3 通过提供一种方法来识别能力供给中的**缺口或重复**，支持不同 COI 中使用的**能力审计过程和类似流程**。
>
> CV-3 指示能力**增量（increments）**——当增量与能力交付相关联时，应将其与采购项目中的**交付里程碑**关联。

官方定义的预期用途：
1. **能力规划**（构建能力阶段规划本身）
2. **能力集成规划**
3. **能力差距分析**

### 2.2 推荐表达形式

| 形式 | 适用场景 | 备注 |
|------|---------|------|
| **表格（能力 × 阶段）** | ⭐ 默认形式 | 行=能力(来源CV-2), 列=阶段(来源CV-1) |
| **甘特图式色带** | 可视化时间跨度 | 跨多阶段的能力用长色条表示 |
| **泳道图** | 多维度对比 | 按组织/域分泳道 |

#### 表格表达规范

| 交叉单元格内容 | 含义 |
|---------------|------|
| 能力增量描述 | 该阶段内此能力的变化 |
| 彩色编码长条 | 能力跨越多个时期的持续可用性 |
| **空白** | ⚠️ 该时期无计划满足该能力需求 |

### 2.3 关键数据元素（DM2 映射）

| DM2 概念 | 在 CV-3 中的角色 |
|---------|-----------------|
| **Capability** | 行维度（来自 CV-2） |
| **Phase / Time Period** | 列维度（来自 CV-1） |
| **Capability Increment** | 单元格内容——某阶段内的能力变化 |
| **Project Milestone** | 能力增量的交付节点（关联 PV-2） |
| **Activity / Condition / Effect / Rule / Measure / Resource Flow** | 构成能力增量的细节要素 |

### 2.4 完整细节

#### 2.4.1 数据填充方法

CV-3 的数据通过以下方式填充：

**方法一：从项目数据推导**
1. 分析**项目数据**确定哪些项目何时交付/升级/撤回能力元素
2. 此数据可部分由 **PV-2 Project Timelines** 提供
3. 将识别出的能力增量按 **CV-2 分类法**结构化
4. 按 **CV-1 阶段**排列

**方法二：从期望能力反推**
1. 定义期望的能力增量集
2. 与项目计划对比
3. 实践中通常**两种方法迭代进行**

#### 2.4.2 变体形式：包含项目名称的 CV-3

> 一个变体 CV-3 可以包含能够交付**能力增量的项目名称**。这种变体可以识别能力缺口和短缺。

**本质**：项目 → 能力 → 时间 三维关系矩阵

这种变体可用于：
- 识别需要**干预的项目**（以填补能力缺口）
- 表示当前计划（按交付时间表显示能力可用性）

#### 2.4.3 核心分析功能

| 功能 | 操作方式 |
|------|---------|
| **差距检测** | 查找空白单元格——某时段无计划满足的能力需求 |
| **冗余检测** | 查找重叠条目——多个能力覆盖同一功能 |
| **增量追踪** | 追踪每阶段新增/升级/撤回的能力变化 |
