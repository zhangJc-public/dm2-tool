# SV-9: Systems Technology & Skills Forecast（系统技术与技能预测）

## 一、定位与目的

### 1.1 这份文档回答什么问题？

**一句话**：未来有哪些新技术和技能会影响我们的架构？它们什么时候成熟？

SV-9 是系统视点的**前瞻雷达**——扫描即将到来的技术变革和人力资源趋势。

核心问题集：
| 问题 | 回答 |
|------|------|
| 未来 1-3 年有什么新技术可用？ | 技术趋势清单 |
| 这些技术什么时候达到可用/成熟？ | 就绪时间预测 |
| 我们需要什么样的技能来驾驭这些技术？ | HR 技能需求 |
| 新技术对现有架构的影响？ | 影响评估 |

## 二、核心内容解析

### 2.1 预测内容

SV-9 总结关于技术和人员趋势的预测，包括：

| 类别 | 内容 |
|------|------|
| **新兴能力** | 新技术带来的新可能性 |
| **行业趋势** | 市场和技术方向的宏观走向 |
| **可用性预测** | 特定硬件/软件服务的就绪时间（附**置信因子**） |
| **当前及未来技能** | 所需的人力资源能力 |

### 2.2 时间框架

> 鉴于本模型的面向未来的性质，预测通常以**短、中、长期时间框架**进行，例如 **6、12 和 18 个月间隔**。

### 2.3 与 StdV-2 的因果关系 ⭐

规范揭示了一个关键的依赖链：

> **某项技术可能依赖于某项 StdV-2 标准**（即 StdV-2 中列出的标准可能在某项技术可用之前不会被采纳）。

这意味着：
```
SV-9 (技术就绪预测)
    ↓ 使能
StdV-2 (标准采纳预测)
    ↓ 驱动
SV-8 (系统演进计划)
    ↓ 影响
架构设计决策
```

### 2.4 与架构过渡计划的关联

> 预测包括对当前架构的**潜在影响**，从而影响**过渡和目标架构**的开发。

SV-9 直接服务于：
- 新能力插入规划
- 现有资源的升级
- 人员再培训需求

### 2.5 表达形式

| 形式 | 说明 |
|------|------|
| **表格** | 时间×技术的矩阵 |
| **时间线** | 可视化技术演进路径 |
| **鱼骨图（Herringbone Diagram）** | 因果分析格式 |

### 2.6 技术与 HR 分开

> 架构师可以为**技术和人力资源**分别生成独立的 SV-9 产品。

所选的具体时间段和追踪的趋势可与架构过渡计划（SvcV-8 可支持）协调。
