# SvcV-6: Services Resource Flow Matrix（服务资源流矩阵）

## 一、定位与目的

### 1.1 这个矩阵回答什么问题？

**一句话**：服务之间具体交换了什么数据？格式、性能、安全属性是什么？

SvcV-6 是 **OV-3 和 SV-6 在服务侧的等价物**——服务间资源流的物理属性详情。

核心问题集：
| 问题 | 回答 |
|------|------|
| 服务间交换的具体数据元素？ | 数据清单 |
| 交换的周期/及时性/吞吐量？ | 性能属性 |
| 信息保障和安全特征？ | 安全属性 |
| 数据格式和媒体类型？ | 技术规格 |

### 1.2 三层资源流矩阵对照 ⭐⭐

| 维度 | **OV-3** (作战逻辑) | **SV-6** (系统物理) | **SvcV-6** (服务物理) |
|------|---------------------|---------------------|----------------------|
| **层级** | 作战层 | 系统层 | **服务层** |
| **交换方** | 作战活动/位置 | **系统** | **服务** |
| **性质** | 逻辑 | 物理 | **物理** |
| **焦点** | 跨能力边界的流 | 跨系统边界的流 | **跨服务边界的流** |
| **非自动化** | 概念性包含 | 也捕获 | 也捕获 |

> **SvcV-6 是 OV-3 的物理等价物**，提供实现 OV-3 中规定的资源流交换的**服务连接详细信息**。

---

## 二、核心内容解析

### 2.1 面向服务的特殊性 ⭐

规范强调了 Net-Centric（面向网络/以网为中心）的实现背景：

> 根据 Net-Centric Data Strategy，面向网络的实现需要关注**服务资源流中的数据**，以及**产生或消费该数据的服务**。
>
> 在面向网络的实现中，并非所有消费者都是已知的，因此本模型强调聚焦于**生产者和资源流**。

这是 SOA 架构的关键特征——**生产者注册、消费者发现**的模式。

### 2.2 核心内容维度

SvcV-6 聚焦于服务特定的交换影响方面：

| 属性类别 | 具体属性 |
|---------|---------|
| **时间属性** | Periodicity / Timeliness / Throughput / Size |
| **安全属性** | Information Assurance / Security Characteristics |
| **数据属性** | Format / Media Type / Accuracy / Units / Applicable Standards |
| **物理引用** | DIV-3 Physical Data Models |

### 2.3 追溯纪律（与 SV-6 相同要求）

> 每个 SvcV-6 表中列出的**服务资源流交换应可追溯到至少一个** OV-3 中的作战资源流，而后者又追溯到 OV-2。

完整追溯链：
```
OV-2(Needline) → OV-3(逻辑流+属性)
                            ↓ 物理化(两条路径)
                    ┌───────┴───────┐
                SV-6(系统流)   SvcV-6(服务流) ← 本视图
                    ↓               ↓
                DIV-3(物理数据)   DIV-3
```

### 2.4 与 SvcV-4 的关系

> SvcV-6 中列出的数据元素与 SvcV-4 中产生/消耗的数据流**不必一一对应**。
>
> 因为 **SvcV-4 更多是逻辑解决方案，而 SvcV-6 是更物理的解决方案**。

### 2.5 内部流不显示

> 同一服务内执行的服务函数之间的资源流**不需要**显示在 SvcV-6 矩阵中。
>
> SvcV-6 关注的是跨**服务边界**的流。

### 2.6 与 SvcV-7 的协作

> **SvcV-7 Services Measures Matrix 基于 SvcV-6 构建，应同时开发**。

## 三、关联视图

| 上游依赖 | 下游支撑 | 同级互补 |
|----------|---------|---------|
| **OV-3**（逻辑流→服务实现） | → DIV-3（物理格式） | OV-3（逻辑 vs 服务物理） |
| **SvcV-2**（连接规格） | → SvcV-7（度量基础） | SV-6（系统 vs 服务物理） |
| SvcV-4（服务函数） | → | |

### 资源流矩阵的完整四象限

```
            逻辑(L)        物理(P)
         ┌──────────┬──────────┐
作战(OV): │  OV-3     │          │
         │ (N:1映射) │          │
系统(SV): │           │  SV-6    │
         │           │ (系统边界)│
服务(S):  │           │  SvcV-6  │ ← 本视图
         │           │ (服务边界)│
         └──────────┴──────────┘
```
