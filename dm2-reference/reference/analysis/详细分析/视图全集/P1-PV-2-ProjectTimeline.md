# PV-2: Project Timeline（项目时间线）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：每个项目什么时候开始、什么时候交付？

PV-2 是项目视点的**进度规划器**——展示所有项目的时间排列和里程碑。

核心问题集：
| 问题 | 回答 |
|------|------|
| 项目 A 何时启动？ | 开始日期 |
| 能力 X 何时到位？ | 交付里程碑 |
| 哪些项目可以并行？ | 时间重叠分析 |
| 关键路径是什么？ | 依赖链上的瓶颈项目 |

### 1.2 核心角色

> PV-2 是**多个数据源的时间协调器**。

PV-2 的信息被广泛引用：
- **CV-5** 用它确定能力向组织的部署时机
- **SV-8/SvcV-8** 用它驱动系统/服务演进描述
- **StdV-2/SV-9/SvcV-9** 将标准/技术预测与其时间阶段关联

## 二、核心内容解析

### 2.1 表达形式

| 形式 | 说明 |
|------|------|
| **甘特图** | ⭐ 最常用：时间×任务条形图 |
| **里程碑列表** | 关键节点的时间戳 |
| **时间线图** | 简化版甘特图 |

### 2.2 关键要素

| 要素 | 说明 |
|------|------|
| **项目名称** | 与 PV-1 一致 |
| **起始/终止时间** | 计划的起止点 |
| **里程碑** | 关键交付节点 |
| **依赖关系** | 项目间的先后约束 |
| **能力交付节点** | 特定能力的可用时间点 |
