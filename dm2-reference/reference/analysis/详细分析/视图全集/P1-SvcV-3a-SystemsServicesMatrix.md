# SvcV-3a: Systems-Services Matrix（系统-服务矩阵）

## 一、定位与目的

### 1.1 这个矩阵回答什么问题？

**一句话**：哪些系统和哪些服务有交互？全局概览。

SvcV-3a 是 SvcV-1 的**表格摘要**——快速展示所有系统与服务之间的交互关系。

核心问题集：
| 问题 | 回答 |
|------|------|
| 系统 X 用到了哪些服务？ | 系统→服务消费关系 |
| 服务 Y 被哪些系统使用？ | 服务→系统消费者 |
| 整体的系统-服务耦合度如何？ | 公共性/冗余评估 |

### 1.2 核心角色

> SvcV-3a 提供 SvcV-1 中指定的**系统和服务交互的表格化摘要**。
>
> 此模型对于**正在过渡到提供服务**的现有系统特别有用。

## 二、核心内容解析

### 2.1 矩阵结构

```
         Serv-A  Serv-B  Serv-C  Serv-D
Sys-1  [  ✓      ✓             ]
Sys-2  [         ✓      ✓     ]
Sys-3  [  ✓              ✓     ]
```

### 2.2 单元格编码（与 SV-3 相同模式）

| 编码维度 | 示例 |
|---------|------|
| **状态** | 现有 / 计划 / 潜在 / 已停用 |
| **关键接口** | 高亮标记 |
| **类别** | 指控 / 情报 / 人事 / 后勤 |
| **密级** | Restricted ~ Top Secret |
| **通信手段** | Rim Loop / Scalable Loop 等 |

> DoDAF 不规定符号。使用符号时需提供图例。

### 2.3 组织方式（同 SV-3）

可有多个 SvcV-3a，按以下方式组织：
- **按域**
- **按作战使命阶段**
- **按解决方案选项**

### 2.4 实践价值

| 场景 | 价值 |
|------|------|
| **遗留系统向服务化迁移** | 追踪迁移进度和覆盖范围 |
| **管理解决方案演进** | 发现插入点和变更面 |
| **新技术/功能插入** | 影响范围评估 |
| **互操作特性比较** | 方案对比 |

## 三、关联视图

| 上游依赖 | 下游支撑 | 同级互补 |
|----------|---------|---------|
| **SvcV-1**（服务上下文→摘要） | → 接口管理 | SV-3（系统侧版本） |
| SvcV-2（资源流详情） | → 冗余检测 | SvcV-3b（服务-服务版本） |

### SV-3 vs SvcV-3a vs SvcV-3b 三件套

```
SV-3:    系统×系统    (系统间接口拓扑)
SvcV-3a: 系统×服务    (系统如何消费服务)  ← 本视图
SvcV-3b: 服务×服务    (服务间协作拓扑)
```
