# CV-7: Capability to Services Mapping（能力到服务映射）

## 一、定位与目的

### 1.1 这张表回答什么问题？

**一句话**：每项服务支撑了什么能力？

CV-7 是**能力视点和服务视点之间的桥梁**——建立从"要什么能力"到"用什么服务"的追溯链路。

核心问题集：
| 问题 | 回答 |
|------|------|
| 这个服务支撑了哪些能力？ | 服务→能力贡献矩阵 |
| 这项能力由哪些服务支撑？ | 服务组合分析 |
| 有没有能力缺少服务支撑？ | 服务缺口识别 |
| 服务是否完全满足能力需求？ | 完全/部分满足判断 |

## 二、核心内容解析

### 2.1 必要性说明

> CV-7 描述**所需能力与使能这些能力的服务之间的映射**。
>
> 确保服务与所需能力匹配至关重要。

CV-7 与 CV-6 是**对称的双子视图**：

| 维度 | CV-6 | CV-7 |
|------|------|------|
| **映射对象** | 能力 → **作战活动** | 能力 → **服务** |
| **右侧视点** | OV（作战视点） | SvcV（服务视点） |
| **回答问题** | "能力靠什么活动实现？" | "能力靠什么服务支撑？" |
| **战略时机** | 战略架构阶段（活动未细化时） | 战略架构阶段（服务未细化时） |

### 2.2 CV-7 vs SV-5a vs CV-6 对比 ⭐

三者的关系形成完整的**追溯链**：

```
能力(CV) ──[CV-6]──→ 作战活动(OV) ──[SV-5a]──→ 系统功能(SV)
    │
    └──[CV-7]──→ 服务(SvcV) ──[SvcV-5]──→ 作战活动(OV)  [交叉验证]
```

| 维度 | CV-7 | SV-5a | CV-6 |
|------|------|-------|------|
| **左端** | 能力 | 作战活动 | 能力 |
| **右端** | 服务 | 系统函数 | 作战活动 |
| **跨越领域** | 能力→服务 | 作战→系统 | 能力→作战 |
| **功能类比** | "能力的服务支撑" | "活动的系统实现" | "能力的活动载体" |

> CV-7 类似于 **SV-5a Operational Activity to System Function Traceability Matrix** —— 但它提供的是**能力模型和服务模型之间的接口**。

### 2.3 映射状态语义

与 CV-6 一致，支持两种满足程度：

| 状态 | 含义 |
|------|------|
| **完全满足** | 服务完整覆盖能力需求 |
| **部分满足** | 服务仅部分覆盖能力需求 |

### 2.4 表达形式

> CV-7 采用**表格呈现**：行=能力，列=服务。

| 标记 | 含义 |
|------|------|
| **X / ✅** | 该能力可用于支撑此服务 |
| **空白** | 无关联 |
| **日期/阶段** | 该能力在该日期/阶段开始支撑此服务 |

### 2.5 版本管理

与 CV-6 相同的原则：

| 策略 | 条件 |
|------|------|
| **单表** | 高层能力 + 标准服务集，关系稳定 |
| **多表分阶段** | 不同阶段差异显著 |

> 如果能力是通用的（见 CV-1），它们与标准服务集之间的关系**不太可能随时间变化**。

### 2.6 战略架构阶段的特殊处理

如果在**服务模型尚未创建**时制作 CV-7（即战略架构早期）：

- ✅ 建议：使用**通用功能（Common Functions）**作为服务占位符
- 这样后续可与正式 SvcV 模型无缝衔接
