# CV-5: Capability to Organizational Development Mapping（能力到组织发展映射）

## 一、定位与目的

### 1.1 这张表回答什么问题？

**一句话**：哪些能力部署到了哪个组织？什么时候到位？

CV-5 是能力视点的**部署规划图**——展示特定阶段下能力向具体组织的分配情况。

核心问题集：
| 问题 | 回答 |
|------|------|
| 这个阶段各组织拥有什么能力？ | 能力-组织的分配矩阵 |
| 能力何时部署到位？ | 按 PV-2 项目时间线 |
| 组织之间如何通过能力交互？ | SV-1/SvcV-1 接口叠加 |
| 有没有能力冗余/重叠/缺口？ | 冗余/重叠/缺口分析 |

### 1.2 在架构描述中的角色

```
┌──────────────────────────────────────────────┐
│      CV-5: 能力-组织发展映射                   │
│     （部署规划器 / 列装时间表）                │
├──────────────────────────────────────────────┤
│                                              │
│           组织A   组织B   组织C   组织D       │
│         ┌──────┬──────┬──────┬──────┐        │
│ 能力X   │  ✅  │      │  ✅  │      │        │
│ 能力Y   │      │  ✅  │  ✅  │  ✅  │        │
```

## 二、核心内容解析

### 2.1 必要性说明

> CV-5 解决**能力需求的履行**问题。它显示**特定阶段**下的规划能力部署和互连，应提供比 CV-3 更详细的依赖分析。

核心用途：
- **Fielding Planning（列装规划）**：能力何时部署到哪个组织
- **Capability Integration Planning（能力集成规划）**
- **Capability Options Analysis（能力选项分析）**
- **Redundancy/Overlap/Gap Analysis（冗余/重叠/缺口分析）**
- **Deployment Level Shortfall Identification（部署级别缺口识别）**

### 2.2 与 CV-3 的关系 ⭐

| 维度 | CV-3（能力阶段规划） | CV-5（组织映射） |
|------|---------------------|-----------------|
| **维度** | 时间线（何时） | 组织×能力矩阵（谁+何时） |
| **粒度** | 阶段级别的粗粒度 | 具体组织级别的细粒度 |
| **焦点** | 能力的成熟时间 | 能力的部署位置 |
| **关系** | CV-3 提供时间框架 | CV-5 放大 CV-3 的某个时间切片 |

> CV-5 + SV-8/SvcV-8（演进描述） + PV-2（项目时间线） = **放大 CV-3 中的信息**。

### 2.3 多阶段模型 ⚠️

**重要**：CV-5 是**按阶段的**——每个阶段可以创建不同的 CV-5：

- 一个架构描述可能需要 **N 个 CV-5**（每阶段一个）
- 各 CV-5 分别表示，但**同一能力可以出现在多个阶段的不同组织中**
- 数据来源：PV-2（项目时间线）、CV-2（分类）、OV-4（组织关系）、SV-1/SvcV-1（接口）

### 2.4 表达形式

> CV-5 通常基于**表格表示**，一轴为组织结构，另一轴为能力。

| 形式 | 适用场景 |
|------|---------|
| **二维矩阵**（⭐ 默认） | 行=能力，列=组织；单元格标记部署状态 |
| **热力图矩阵** | 用颜色深浅表示能力成熟度或覆盖程度 |
| **泳道图** | 按组织分泳道，展示能力和交互流 |

当能力跨多个组织部署时，可创建**父级组织**作为上下文容器。

### 2.5 关键实践指南 ⭐⭐

规范给出了非常重要的时机指导：

❌ **不应**在开发能力/用户需求时制作 CV-5 —— 这会**限制解决方案空间**
✅ **应在方案确定后**再制作 CV-5 —— 此时作为**项目管理层面的参考信息**

> "CV-5 应被视为**能力交付进度表的摘要**（因此有人认为它属于 PV 视点）。"

这意味着 CV-5 处于**能力视点和项目视点的边界地带**。

### 2.6 完整细节

#### 数据来源

| 来源视图 | 提供 |
|---------|------|
| **PV-2 Project Timelines** | 能力交付时间点 |
| **CV-2 Capability Taxonomy** | 能力定义和分类 |
| **OV-4 Organizational Relationships** | 组织结构 |
| **SV-1 Systems Interface Description** | 系统交互（可选叠加） |
| **SvcV-1 Services Context Description** | 服务交互（可选叠加） |

#### 图形设计原则

- **不要过度填充**：避免图中能力和组织过多导致不可读
- CV-5 本质上是**摘要视图**，不是详细规格

## 三、关联视图

| 上游依赖（输入） | 下游支撑（输出） | 同级互补 |
|------------------|-----------------|---------|
| CV-2（能力定义） | → 列装规划执行 | CV-3（时间框架） |
| CV-3（阶段划分） | → 缺口分析 | CV-4（依赖关系） |
| OV-4（组织结构） | → 能力审计 | PV-2（项目时间线，强依赖） |
| PV-2（时间线） | → | SV-8/SvcV-8（演进） |
| SV-1/SvcV-1（接口） | → | |

### CV-5 在能力→项目链条中的位置

```
战略层: CV-1(愿景) → CV-2(分类)
                         ↓
战术层: CV-4(依赖) → CV-3(时间线)
                         ↓ 放大
执行层: CV-5(组织映射) ──┐
                         ├─→ PV-1(项目组合)
                         ├─→ PV-2(时间线) ← 数据来源!
                         └─→ PV-3(项目-能力映射)
```
