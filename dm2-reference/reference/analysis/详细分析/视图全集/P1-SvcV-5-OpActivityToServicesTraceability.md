# SvcV-5: Operational Activity to Services Traceability Matrix（作战活动-服务追溯矩阵）

## 一、定位与目的

### 1.1 这个矩阵回答什么问题？

**一句话**：每个作战活动由哪些服务来支撑？

SvcV-5 是**作战活动到服务函数的追溯桥梁**——与 CV-7 和 SV-5 形成完整的追溯链。

核心问题集：
| 问题 | 回答 |
|------|------|
| 活动 X 由哪些服务支撑？ | 活动→服务映射 |
| 服务 Y 支持了哪些活动？ | 服务覆盖范围 |
| 有没有活动缺少服务支撑？ | 缺口识别 |

### 1.2 在追溯链中的位置 ⭐

```
能力侧:  CV-7 (能力→服务)
           ↓
作战侧:  OV-5b (活动模型)
           ↓ ═══ SvcV-5 追溯桥 ═══
服务侧:  SvcV-5 (活动→服务函数)  ← 本视图
           ↓
       SvcV-4 (服务功能模型)
```

### 1.3 与 SV-5a 的类比

> 功能上类似于 **SV-5a**（将系统函数映射到作战活动），但提供的是**能力和服务模型之间**的接口。

## 二、核心内容解析

### 2.1 状态指示器（交通灯）

与 SV-5a/SV-5b 相同的状态体系：

| 颜色 | 含义 |
|------|------|
| 🔴 Red | 功能已规划但未开发 |
| 🟡 Yellow | 部分功能已提供（或完整功能未部署） |
| 🟢 Green | 完整功能已部署 |
| ⬜ Blank | 无服务支撑 / 无关联 |

> 发布时应注明日期。

### 2.2 表达形式

- 行 = Operational Activities（来自 OV-5a/b）
- 列 = Service Functions（来自 SvcV-4）
- 单元格 = X / 日期 / 阶段

## 三、关联视图

| 上游依赖 | 下游支撑 | 同级互补 |
|----------|---------|---------|
| OV-5a/b（活动） | → 缺口分析 | SV-5a（系统侧镜像） |
| SvcV-4（服务函数） | → 能力审计 | CV-7（能力侧镜像） |

### 完整的活动追溯三胞胎

```
活动(OV) ──[CV-6]──→ 能力(CV)  ← 战略桥
活动(OV) ──[SV-5a]──→ 系统(SV)  ← 系统桥
活动(OV) ──[SvcV-5]──→ 服务(SvcV) ← 服务桥  ← 本视图
```
