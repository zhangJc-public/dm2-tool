# CV-2: Capability Taxonomy（能力分类法）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：我们拥有/需要**哪些能力**？它们之间是什么层级关系？

CV-2 是**整个 DoDAF 能力体系的核心数据结构**——一个树形的能力层级，定义了架构中引用的所有能力的完整清单和分类关系。

核心问题集：
| 问题 | 回答 |
|------|------|
| 这个架构涉及哪些能力？ | 完整能力列表 |
| 这些能力如何组织？ | 层级分类结构 |
| 现状 vs 目标有什么变化？ | 时间线上的能力差异 |
| 每个能力的性能要求是什么？ | 度量指标 + 阈值条件 |

### 1.2 在架构描述中的角色

```
┌──────────────────────────────────────────────┐
│           CV-2: 能力分类法                    │
│      （能力体系的"字典" / 全局引用源）         │
├──────────────────────────────────────────────┤
│                                              │
│   根节点: 国防 / 企业级能力                    │
│    ├── 战斗能力                               │
│    │   ├── 机动                              │
│    │   │   ├── 地面机动                       │
│    │   │   └── 空中机动                       │

│    │   └── 决策协同                          │
```

**关键角色**：
- **全局能力注册表**——所有其他视点引用能力的权威来源
- **能力审计的基础**——对照此表检查能力覆盖度
- **需求推导的源头**——高层用例和用户需求的来源文档
- **AV-2 的子集**——CV-2 是 AV-2 中 Capabilities 部分的结构化视图

### 与 AV-2 的关键区别

| 特征 | AV-2 (集成词典) | CV-2 (能力分类法) |
|------|----------------|-------------------|
| 范围 | 所有术语 | 仅能力相关 |
| 关系类型 | 多种 | **仅 sub-supertype**（纯特化） |
| 用途 | 一致性基础 + 追溯 | 能力规划 + 审计 + 差距分析 |
| 结构 | 层级 + 元数据 | 严格层级 |

## 二、核心内容解析

### 2.1 必要性说明（官方 Purpose 翻译）

> CV-2 **捕获能力分类法**。该模型呈现能力的**层级结构**。这些能力可以在**时间线上下文**中展示——即它可以显示当前能力和未来所需的能力。
>
> CV-2 指定在**一个或多个架构中被引用的所有能力**。此外，它可以用作开发高层用例和用户需求的**来源文档**。

官方定义的预期用途：
1. **识别能力需求**
2. **能力规划**（构建能力分类法本身）
3. **编码必需的能力要素**
4. **能力审计**
5. **能力差距分析**
6. **内聚性用户需求集的推导来源**
7. **为架构提供参考能力**

### 2.2 推荐表达形式

| 形式 | 适用场景 | 备注 |
|------|---------|------|
| **层级树形图** | ⭐ 默认形式 | 根=最通用, 叶=最具体 |
| 缩进文本列表 | 简单场景 | 轻量级表达 |
| 表格（带属性列） | 需要附加度量时 | 叶节点的属性可作为附录 |
| 图形化工具输出 | 大型复杂分类法 | 支持展开/折叠、搜索 |

> ⚠️ CV-2 **没有强制规定的结构**，只要能支持结构化/层级列表的表示即可。

### 2.3 关键数据元素（DM2 映射）

| DM2 概念 | 在 CV-2 中的角色 | 必需性 |
|---------|-----------------|--------|
| **Capability（能力）** | 核心实体 | ✅ 必须 |
| **Sub-supertype Relation** | 层级关系的唯一类型 | ✅ 必须 |
| **Measure（度量）** | 叶节点性能指标 | ⚠️ 推荐 |
| **Condition（条件）** | 度量的适用环境 | ⚠️ 推荐 |
| **Phase（阶段）** | 能力的时间维度 | 可选 |

#### 能力属性的量化维度

官方明确指出需要包含的定量属性示例：
- 处理速度（speed of processing）
- 推进速率（rate of advance）
- 最大探测范围（maximum detection range）

这些属性值可以：
- 关联到特定阶段
- 或标记为 "As-Is"（现状）/"To-Be"（目标）目标

### 2.4 完整细节

#### 2.4.1 结构约束

**唯一的关系类型：sub-supertype**

> 与 AV-2 不同，CV-2 只使用一种特化关系：**sub-supertype**（子-超类型）。这是一种两个类之间的关系，其中第二个是第一个的**纯特化（pure specialization）**。

这意味着：
- ❌ 不允许"部分-整体"（part-whole）关系混入
- ❌ 不允许"关联-引用"关系混入
- ✅ 只有严格的 "is-a-kind-of" 继承关系
- 根节点 = 最通用的能力类别
- 叶节点 = 最具体的能力项

#### 2.4.2 时间维度的处理

> 在 DoDAF V2.0 中，能力存在于**空间和时间上**——即它们旨在为正在建模的企业**整个生命周期**提供一个框架。

这意味着：
1. **同一个 CV-2 可以适用于所有架构阶段**（现状、目标、过渡）
2. 能力可以在时间线上标注其存在状态
3. 区分"当前已具备"vs"未来需要"的能力

#### 2.4.3 能力粒度的灵活性

> 当能力被引用到作战或系统架构中时，可能某个设施、位置、组织或配置满足**超过一个层级的能力**。

这说明：
- CV-2 的层级是**逻辑分类**，不是物理分解
- 同一底层资源可能同时支撑多个层级的抽象能力

## 三、关联视图

| 上游依赖（输入） | 下游支撑（输出） | 同级互补 |
|------------------|-----------------|---------|
| CV-1（愿景范围） | ← CV-3（阶段的行来源） | CV-3（时间维度） |

| | → CV-7（能力→服务映射的目标端） | |
| | → OV/SV/SvcV 各视图的能力引用源 | |

### CV-2 作为"能力线程"的核心

> **Capability Thread（能力线程）** = 对某个特定能力运行查询的结果。
>
> 例如，如果架构包含作战活动、规则和系统，那么能力线程就等同于与该特定能力相关的那些活动、规则和系统。

CV-2 是能力线程的**起点**——没有它，无法定义"查询"的对象。
