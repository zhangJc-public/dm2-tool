# CV-6: Capability to Operational Activities Mapping（能力到作战活动映射）

## 一、定位与目的

### 1.1 这张表回答什么问题？

**一句话**：每个作战活动需要什么能力支撑？

CV-6 是**能力视点和作战视点之间的桥梁**——建立从"要什么能力"到"做什么活动"的追溯链路。

核心问题集：
| 问题 | 回答 |
|------|------|
| 这个活动需要哪些能力？ | 活动→能力支撑矩阵 |
| 这项能力支撑了哪些活动？ | 能力使用范围 |
| 有没有活动缺少能力支撑？ | 能力缺口识别 |
| 能力是否完全满足了活动需求？ | 完全满足 / 部分满足 / 不满足 |

## 二、核心内容解析

### 2.1 必要性说明

> CV-6 描述**所需能力与使能这些能力的活动之间的映射**。
>
> 确保作战活动与所需能力匹配至关重要。

核心定位：
- **桥梁角色**：连接 CV 系列分析的能力 ↔ OV 系列分析的作战活动
- **追溯功能**：从能力需求追溯到具体活动
- **审计功能**：检查是否有活动未被能力覆盖

### 2.2 CV-6 vs SV-5a 的类比 ⭐

规范明确指出两者的相似性和区别：

| 维度 | CV-6 | SV-5a |
|------|------|-------|
| **左侧** | 能力（Capability） | 作战活动（Operational Activity） |
| **右侧** | 作战活动（Operational Activity） | 系统函数（System Function） |
| **跨越** | **能力 → 作战** | **作战 → 系统** |
| **本质** | 战略到战术的接口 | 战术到实现的接口 |
| **类比** | "我们要的能力靠什么活动来实现" | "这些活动由什么系统功能来支撑" |

> CV-6 类似于 **SV-5a Operational Activity to Systems Function Traceability Matrix** —— 但它提供的是**能力和作战模型之间的接口**，而非作战和系统模型之间。

### 2.3 映射状态的语义

CV-6 明确支持**两种满足程度**：

| 状态 | 含义 | 示例 |
|------|------|------|
| **完全满足** | 活动完全实现了能力要求 | "目标识别"活动完整支撑"态势感知"能力 |
| **部分满足** | 活动只部分覆盖了能力需求 | "手动情报"活动仅部分支撑"全面感知"能力 |

这一区分对于**能力缺口分析**至关重要——部分满足不等于没有，但也不等于足够。

### 2.4 表达形式

> CV-6 可以采用**表格呈现**。

| 形式 | 说明 |
|------|------|
| **矩阵表**（⭐ 默认） | 行=能力，列=活动 |
| **日期/阶段标注** | 单元格中填写日期或阶段号，而非简单打勾 |

**推荐做法**：
- 如果在**战略架构阶段**（作战模型尚未创建）制作 CV-6：
  - 建议使用**通用功能（Common Functions）**作为活动占位符
- 这样后续可以无缝对接正式的 OV 模型

### 2.5 版本管理

原则上可以为**每个能力开发阶段**或**每种阶段规划场景**创建不同的 CV-6：

| 策略 | 适用条件 |
|------|---------|
| **单表通用** | 高层能力 + 标准活动集，关系稳定不变 |
| **多表分阶段** | 不同阶段的能力/活动差异显著 |

如果能力是通用的（见 CV-1 Vision），它们与标准活动集之间的关系通常**不会随时间变化**。

### 2.6 用途

| 用途 | 说明 |
|------|------|
| **能力需求追溯** | 从能力追踪到支撑它的作战活动 |
| **能力审计** | 检查能力覆盖完整性 |
| **缺口标识** | 发现无活动支撑的能力需求 |
