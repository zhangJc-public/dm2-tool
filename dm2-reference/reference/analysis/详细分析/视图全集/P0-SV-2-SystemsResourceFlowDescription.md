# SV-2: Systems Resource Flow Description（系统资源流描述）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：系统之间的接口**精确规格是什么**？（协议、端口、连接方式）

SV-2 是 SV-1 的**详细化版本**——从"有哪些接口"到"每个接口的精确定义"，包括协议栈和物理/逻辑连接。

核心问题集：
| 问题 | 回答 |
|------|------|
| 哪些端口互相连接？ | 端口连接矩阵 |
| 连接使用什么协议？ | 协议栈定义 |
| 物理还是逻辑连接？ | 连接类型说明 |
| 接口是否已存在？ | 现有 vs 待建 |

### 1.2 在架构描述中的角色

```
┌──────────────────────────────────────────────┐
│        SV-2: 系统资源流描述                   │
│   (SV-1 的精确规格 / 接口级细节)             │
├──────────────────────────────────────────────┤
│                                              │
│   SV-1 (概览) ──细化──→ SV-2 (精确规格)      │
│                                              │
│   SV-2 包含:                                  │
│   ├── Systems（系统列表）                     │

└──────────────────────────────────────────────┘
```

## 二、核心内容解析

### 2.1 必要性说明

> SV-2 **指定系统之间的 System Resource Flows**，也可列出连接中使用的**协议栈**。
>
> SV-2 DoDAF 模型用于给出系统之间连接的**精确规格**。这可以是现有连接，或待建立连接的规格。

预期用途：
- **资源流规格**

### 2.2 推荐表达形式

| 形式 | 适用场景 | 说明 |
|------|---------|------|
| **接口规格表/图** | ⭐ 默认形式 | 每个资源流一张或汇总 |
| **端口连接图** | 可视化拓扑 | 展示哪些端口互联 |
| 协议栈分层图 | 详细通信设计 | 展示每层使用的协议 |

### 2.3 关键数据元素

| 元素 | 说明 |
|------|------|
| **Systems** | 参与连接的系统实体 |
| **Ports** | 每个系统上的接口端口 |
| **Port Connections** | 哪些端口互相连接 |
| **Resource Flow Definition** | 资源流的物理/逻辑连接性 + 协议 |
| **Protocol Stack** | 连接中使用的完整协议栈 |
| **Network（as System）** | 网络基础设施也作为系统建模 |

### 2.4 完整细节

#### 2.4.1 构建选项

> 架构师可以选择为所有系统的每个 Resource Flow 创建一个图表，或在可能的情况下在一个图上显示所有 Resource Flows。

#### 2.4.2 每个 SV-2 模型可显示：

1. **哪些端口已连接？**
2. **端口所属的系统**
3. **System Resource Flow 的定义**——包括：
   - 物理/逻辑连接性
   - 连接中使用的任何协议

#### 2.4.3 网络建模约定

> ⚠️ **网络被表示为 Systems**。架构师可以选择将其他系统显示为网络的组件（即如果它们是网络基础设施的一部分）。

#### 2.4.4 协议约束

> SV-2 图表中引用的任何协议都需要在 **StdV-1 Standards Profile** 中定义。
