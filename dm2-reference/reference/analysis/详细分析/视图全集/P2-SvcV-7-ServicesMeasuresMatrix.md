## 一、视图概述

### 1.1 定义与目的

```
┌──────────────────────────────────────────────┐
│         SvcV-7: 服务度量矩阵                  │
│     (服务性能指标和质量属性度量)                │
├──────────────────────────────────────────────┤
│                                              │
│  核心问题: "服务的性能表现如何？质量达标吗？"   │
│                                              │
│  ┌──────────┬─────────┬─────────┬──────────┐│
│  │ 服务指标 │ 目标值   │ 当前值   │ 状态     ││
│  ├──────────┼─────────┼─────────┼──────────┤│
```

**目的**: 描述在适当时间框架内，服务模型元素的性能度量（指标）和质量属性。回答"服务的性能表现如何？是否达到预期目标？"

### 1.2 与 SV-7 的关系

| 维度 | SV-7 (系统) | SvcV-7 (服务) |
|------|------------|---------------|
| **度量对象** | 系统、硬件组件 | 服务实例、服务接口 |
| **关注点** | 系统级性能参数 | 服务级 SLA 指标 |
| **典型指标** | 处理器利用率、内存占用 | 响应时间、吞吐量、可用性 |
| **应用场景** | 系统选型评估 | 服务级别协议(SLA)管理 |

| DM2 数据组 | 核心概念 | 在 SvcV-7 中的角色 |
|-----------|---------|-------------------|

## 五、实践指南

### 5.1 适用场景

✅ **建议制作 SvcV-7 的信号**：
- 架构涉及**服务级别协议(SLA)** 管理
- 需要量化证明**服务质量达标**
- 存在**多供应商/多来源**的服务需要对比评估
- 进行**服务选型决策**

❌ **可跳过的情形**：
- 架构仅做概念设计，不涉及详细规格
- 所有服务均为内部开发且无 SLA 要求

### 5.2 制作要点

1. **从 SvcV-2/SvcV-4 中提取被度量的服务清单**
2. **参考 StdV-1 确定行业标准和合规要求**
3. **区分"必须项"(SLA硬指标) 和"期望项"(改进目标)**
4. **定期更新——度量数据具有时效性**
5. **使用红/黄/绿三色状态快速传达健康度**

## 六、中国适配要点

| 中国标准/规范 | 对应关系 |
|-------------|---------|
| **GB/T 28827** (信息技术服务 运行维护) | 服务质量评价体系直接映射到 SvcV-7 指标 |
| **等保 2.0** (安全计算环境) | 安全相关度量指标可作为 SvcV-7 子集 |
| **ITSS 服务标准** | 服务响应时间、解决率等 KPI 直接适用 |
| **信创适配场景** | 国产化替代后需重新基线化所有度量值 |

⚠️ **特别注意**: 在安全架构中，SvcV-7 应包含安全专项指标：
- 身份认证成功率/失败率
- 加解密操作延迟开销
- 审计日志完整性
- 访问控制策略执行准确率
