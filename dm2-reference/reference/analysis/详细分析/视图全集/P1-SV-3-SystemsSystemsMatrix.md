# SV-3: Systems-Systems Matrix（系统-系统矩阵）

## 一、定位与目的

### 1.1 这个矩阵回答什么问题？

**一句话**：哪些系统之间有交互？整体接口拓扑是什么样的？

SV-3 是 SV-1（系统接口描述）的**表格摘要**——快速概览所有系统间的交互关系。

核心问题集：
| 问题 | 回答 |
|------|------|
| 系统 A 和系统 B 有没有接口？ | 有/无判断 |
| 整体系统的互联密度如何？ | 公共性/冗余度评估 |
| 哪些系统是关键连接枢纽？ | 中心性分析 |
| 不同方案的互操作特性如何比较？ | 方案对比 |

### 1.2 在架构描述中的角色

```
SV-1 (图形: 详细接口规格)
    ↓ 提炼为表格摘要
SV-3 (矩阵: 全局交互一览)  ← 本视图
    ↓ 快速评估
├─ 冗余检测 ("这两列完全一样?")
├─ 缺口发现 ("这行全是空的?")
└─ 方案对比 (不同方案的矩阵差异)
```

> SV-3 提供 SV-1 中指定的系统交互的**表格化摘要**，支持对潜在公共性和冗余度的**快速评估**。

## 二、核心内容解析

### 2.1 矩阵结构

> SV-3 通常以**矩阵形式呈现**，系统资源列在行和列上，单元格表示是否存在交互。

```
       Sys-A  Sys-B  Sys-C  Sys-D
Sys-A [  ✓      ✓             ]
Sys-B [  ✓      ✓      ✓     ]
Sys-C [         ✓      ✓     ✓  ]
Sys-D [                ✓      ✓  ]
```

### 2.2 单元格信息编码

规范指出多种类型的交互信息可在单元格中表示：

| 编码维度 | 示例 |
|---------|------|
| **状态** | 现有 / 计划中 / 潜在 / 已停用 |
| **关键接口标记** | 高亮显示核心接口 |
| **类别** | 指控 / 情报 / 人事 / 后勤 |
| **密级** | Restricted / Confidential / Secret / Top Secret |
| **通信手段** | Rim Loop / Scalable Loop 等 |

> DoDAF 不规定使用的符号。如果使用符号，需要提供**图例（Key）**。

### 2.3 组织灵活性

> 根据架构描述的目的，可以有**多个 SV-3**。

SV-3 套件可按多种方式组织：

| 组织方式 | 适用场景 |
|---------|---------|
| **按域** | 多域架构（如陆海空天网） |
| **按作战使命阶段** | 不同阶段的系统配置差异大 |
| **按解决方案选项** | 需要对比多个备选方案 |

### 2.4 实践价值

SV-3 特别适合：

| 场景 | 价值 |
|------|------|
| **管理解决方案演进** | 追踪接口变化趋势 |
| **新技术/功能插入** | 发现插入点 |
| **系统和活动的重新分布** | 配置变更影响面 |
| **方案选项互操作特性比较** | 并排对比多个方案 |
