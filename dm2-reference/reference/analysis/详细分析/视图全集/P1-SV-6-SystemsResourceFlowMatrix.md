# SV-6: Systems Resource Flow Matrix（系统资源流矩阵）

## 一、定位与目的

### 1.1 这个矩阵回答什么问题？

**一句话**：系统之间交换了什么数据？格式、性能、安全属性是什么？

SV-6 是 **OV-3 的物理等价物**——从逻辑资源流深入到系统间的实际数据交换规格。

核心问题集：
| 问题 | 回答 |
|------|------|
| 系统 A 向系统 B 发送了什么数据？ | 具体的数据元素 |
| 数据交换的时效要求？ | Timeliness / Throughput |
| 安全级别和信息保障要求？ | IA / Classification |
| 数据格式和媒体类型？ | Format / Media Type |
| 与作战资源流的追溯？ | → OV-3 Needline |

### 1.2 在架构描述中的角色

```
OV-3 (逻辑资源流矩阵)
    ↓ 物理化实现
SV-6 (系统资源流矩阵)  ← 本视图
    ↓ 数据定义
DIV-3 (物理数据模型)
```

> **SV-6 是逻辑 OV-3 表格的物理等价物**，提供实现 OV-3 中规定的资源流交换的系统连接详细信息。

## 二、核心内容解析

### 2.1 SV-6 vs OV-3 vs SvcV-6 的三层对照 ⭐⭐

| 维度 | **OV-3** (逻辑) | **SV-6** (系统物理) | **SvcV-6** (服务物理) |
|------|------------------|---------------------|----------------------|
| **层级** | 作战层 | 系统层 | 服务层 |
| **交换方** | 作战活动/位置 | **系统** | **服务** |
| **焦点** | 业务语义 | 实现**细节** | 服务契约细节 |
| **性质** | 逻辑 | 物理 | 物理 |
| **追溯方向** | → SV-6 / SvcV-6 | ← OV-3 | ← OV-3 |
| **非自动化流** | 包含概念 | **也捕获**（如口头指令） | 也捕获 |

### 2.2 核心内容要素

SV-6 聚焦于系统特定的资源交换细节：

#### 传输属性

| 属性类别 | 具体属性 |
|---------|---------|
| **时间属性** | Periodicity（周期性）、Timeliness（及时性）、Throughput（吞吐量） |
| **规模属性** | Size（大小） |
| **安全属性** | Information Assurance（信息保障）、Security Characteristics（安全特征） |
| **数据属性** | Format（格式）、Media Type（媒体类型）、Accuracy（精度）、Units（单位） |
| **标准属性** | System Data Standard（系统数据标准）、DIV-3 引用 |

### 2.3 追溯纪律 ⚠️

规范强调建模纪律：

> **需要建模纪律来确保架构模型的连贯性**。
>
> SV-6 表中列出的**每个系统资源流交换都应可追溯到** OV-3 中至少一个作战资源流，而后者又可追溯到 OV-2 中的资源流。

完整的追溯链：
```
OV-2 (Needline)
  → OV-3 (逻辑资源流 + 属性)
    → SV-6 (系统资源流 + 物理属性)  ← 本视图
      → DIV-3 (物理数据格式)
```

### 2.4 范围界定

> SV-6 关注的是**跨系统边界**的流。

重要限制：
- ❌ 同一系统内部执行的**系统函数之间**的数据流**不需要**显示在 SV-6 中
- ✅ SV-6 只展示**跨系统边界的流**
- 如果资源流是信息类的，可能需要在 **Data and Information Models** 中反映

### 2.5 与 SV-4 的关系

> SV-6 中列出的数据元素与 SV-4 中产生/消耗的数据流（输入/输出）**不必一一对应**。
>
> 因为 SV-4 更多是**逻辑解决方案**，而 SV-6 是**更物理的解决方案**。

### 2.6 与 SV-7 的协作

> **SV-7 Systems Measures Matrix 模型建立在 SV-6 之上，应同时开发**。

SV-6 定义"交换什么"，SV-7 定义"交换得怎么样"（度量指标）。

## 三、关联视图

| 上游依赖（输入） | 下游支撑（输出） | 同级互补 |
|------------------|-----------------|---------|
| **OV-3**（逻辑流→物理实现） | → DIV-3（物理数据格式） | OV-3（逻辑 vs 物理） |
| SV-1/SV-4（系统/函数定义） | → SV-7（度量基础） | SvcV-6（系统 vs 服务） |
| DIV-2（逻辑数据） | → 接口规格说明书 | |

### 资源流的四层递进

```
作战层: OV-2(Needline) → OV-3(逻辑资源流+属性)
                                    ↓ 物理化
系统层: SV-1(接口)     → SV-6(系统资源流+物理属性) ← 本视图
                                    ↓ 度量化
度量层: SV-7(系统度量矩阵)

服务层: SvcV-1(上下文) → SvcV-6(服务资源流+物理属性)  [镜像]
                                    ↓
       SvcV-7(服务度量矩阵)
```
