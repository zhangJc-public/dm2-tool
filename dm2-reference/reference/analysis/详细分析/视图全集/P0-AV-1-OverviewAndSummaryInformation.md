# AV-1: Overview and Summary Information

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：这份架构描述是什么？为谁做的、为什么做、包含什么？

AV-1 是整个架构描述的「**封面+目录+执行摘要**」。它让高层决策者能在众多架构描述中快速筛选出需要深入了解的那一份。

核心问题集：
| 问题 | 回答 |
|------|------|
| 这是什么架构？ | 架构描述标识（名称、作者、组织） |
| 为什么做这个？ | 目的与视角 |
| 覆盖什么范围？ | 选了哪些视图模型、时间跨度 |
| 处于什么背景？ | 任务、条令、概念、威胁环境 |
| 进展如何？ | 状态（创建/验证/保障） |
| 用了什么工具？ | 工具套件和文件格式 |

### 1.2 在架构描述中的角色

```
┌─────────────────────────────────────────────┐
│              AV-1: 架构概览                  │
│         （入口 + 导航 + 执行摘要）            │
├─────────────────────────────────────────────┤
│                                             │
│   ┌─────────┐ ┌─────────┐ ┌─────────┐      │
│   │ OV-1    │ │ CV-1    │ │ 其他视图 │      │
│   │ (图形)  │ │ (战略)  │ │ ...     │      │
│   └────┬────┘ └─────────┘ └─────────┘      │
│        │                                   │
│   AV-1 的文字内容描述 OV-1 图形中的概念       │
│                                             │
└─────────────────────────────────────────────┘
```

**关键角色**：
- **架构开发的规划指南**（初始阶段）— 定义范围、参与组织
- **导航辅助**（完成后）— 说明"谁、什么、何时、为何、如何"
- **搜索辅助**（存入仓库后）— 支持在架构库中检索
- **决策沟通**（最终版）— 与 OV-1 图形一起构成面向高层的执行摘要

## 二、核心内容解析

### 2.1 必要性说明（官方 Purpose 翻译）

> AV-1 提供一致的**执行级摘要信息**，允许在不同架构描述之间快速参考和比较。AV-1 的文字内容描述 OV-1 图形表示中所包含的概念。AV-1 为架构描述**框定上下文**，包括可能影响基于架构的工作计划的高层决策的假设、约束和限制。

官方定义的用途（Usage）：
1. **界定架构工作的范围**（Scope）
2. **提供架构工作的上下文**（Context）
3. **定义架构工作本身**（Define）
4. **总结架构工作的发现**（Summarize findings）
5. **协助在架构仓库内搜索**

### 2.2 推荐表达形式

| 形式 | 适用场景 | 备注 |
|------|---------|------|
| **结构化文本** | ⭐ 默认形式 | 通常使用组织自定义模板 |
| 模板表单 | 多项目一致性 | 组织应创建统一模板确保跨项目可比性 |
| OV-1 配图 | 最终版执行摘要 | AV-1 文字 + OV-1 图形 = 完整的高层摘要 |

### 2.3 关键数据元素（DM2 映射）

AV-1 本身是**元数据容器**——它不直接映射到单一 DM2 数据组，而是引用其他所有视点的内容：

| AV-1 章节 | 关联 DM2 概念 | 来源视点 |
|----------|--------------|---------|
| 架构描述标识 | Performer / Project | PV / DM2-Performer |
| 范围 | Capability / Activity | CV / DM2-Activity |
| 目的与视角 | — | 自身定义 |
| 背景（任务/条令/概念） | Activity / Rule / Measure | OV / DM2-Rules |
| 状态 | — | CM 流程 |
| 工具与文件格式 | — | 工具链元数据 |
| 发现与建议 | — | 分析产出 |
| 成本 | Measure | DM2-Measure |

### 2.4 完整细节

#### 2.4.1 核心描述项（Detailed Description）

**① Architectural Description Identification（架构描述标识）**
- 名称、架构师、开发组织
- 假设与约束
- 批准权威机构
- 完成日期
- 所需工作量级别

**② Scope（范围）**
- 已选择和开发的视点、DoDAF 模型、Fit-for-Purpose 视图
- **时间属性**：覆盖的时间框架（具体年份或"现状/目标/过渡"等标记）
- 组织实体和时间线

**③ Purpose and Perspective（目的与视角）**
- 架构描述的需求
- 将要展示的内容
- 将要应用的分析类型
- 谁来分析、做什么决策、预期产生什么行动
- 开发视角

**④ Context（背景）**
- 任务（Mission）
- 条令（Doctrine）
- 相关目标和愿景声明
- 作战概念（Concepts of Operation）
- 场景（Scenarios）
- 信息保障上下文（需保护的系统/服务数据类型、机密等级、威胁环境）
- 其他威胁和环境条件
- 地理区域
- 标准/规则/准则/约定的权威来源
- 与并行架构工作的链接

**⑤ Status（状态）**
- 发布或开发时架构的状态（可能在架构开发之前）
- 涉及创建、验证和保障活动

**⑥ Tools and File Formats Used（工具与文件格式）**
- 开发所用的工具套件
- 架构模型的文件名和格式

**⑦ Assumptions and Constraints（假设与约束）**

**⑧ Architecture Development Schedule（开发进度安排）**
- 开始日期、里程碑、完成日期、其他关键日期
- 更多细节可在 Project Viewpoint 中反映

#### 2.4.2 可选扩展（用于支持分析的架构）

如果架构用于支持分析，AV-1 可扩展包括：

**⑨ Findings（发现）**
- 短缺识别
- 推荐的系统实现方案
- 技术插入机会

**⑩ Costs（成本）**
- 架构预算、成本预测、实际发生成本
- 集成成本、设备成本、其他成本

#### 2.4.3 版本演进策略

| 版本时机 | 重点内容 |
|---------|---------|
| **初始版本** | 聚焦工作范围、文档化范围、涉及的组织 |
| **中间版本** | 在其他模型开发验证后，记录范围调整和其他变更 |
| **最终版本** | 总结发现，面向高层决策者；与 OV-1 一起作为执行摘要 |

最终版的 AV-1 还需要：
- 列出每个模型并提供简短评论
- 说明应用于建模的方法
- 模型分组理由
- 影响各模型的假设

### 与 OV-1 的特殊关系

> AV-1 的文字内容**描述 OV-1 图形中表示的概念**。两者互为补充：
> - **OV-1** = 一张图说清"这是什么架构"（视觉冲击）
> - **AV-1** = 一段文写透"为什么做、怎么做、做到哪了"（信息密度）
