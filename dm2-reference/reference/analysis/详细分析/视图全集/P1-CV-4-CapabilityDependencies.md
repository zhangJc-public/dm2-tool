# CV-4: Capability Dependencies（能力依赖关系）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：能力之间谁依赖谁？哪些能力必须协同工作？

CV-4 是能力视点中的**依赖分析器**——揭示规划能力之间的逻辑关联和分组关系。

核心问题集：
| 问题 | 回答 |
|------|------|
| 哪些能力之间存在依赖？ | 能力间的依赖关系网络 |
| 哪些能力需要一起交付？ | 逻辑分组（集成需求） |
| 某个能力的变更会影响谁？ | 影响分析的基础 |
| 能力之间的依赖类型是什么？ | 可自定义的依赖分类 |

## 二、核心内容解析

### 2.1 必要性说明

> CV-4 描述**规划能力之间的依赖关系**。它还定义能力的**逻辑分组**。

关键定位：
- **依赖分析**：识别能力之间的前置/协同/约束关系
- **逻辑分组**：基于集成需求将相关能力归组
- **企业治理导向**：分组目的是指导企业管理决策

### 2.2 CV-4 vs CV-2 的关键区别 ⭐

这是理解 CV-4 最重要的一点：

| 维度 | CV-2（能力分类法） | CV-4（能力依赖关系） |
|------|-------------------|---------------------|
| **关系类型** | **特化-泛化**（IS-A 继承） | **任意依赖**（前置/协同/约束等） |
| **结构形式** | 树状层级（分类体系） | 网状图（依赖网络） |
| **回答问题** | "能力属于哪一类？" | "谁依赖谁？" |
| **类比** | 生物学的物种分类 | 项目管理的 PERT 图 |
| **唯一性** | 通常只有一份 | 可以按需创建多份 |

> CV-2 只处理 **specialization-generalization 关系**（即能力分类学），而 CV-4 处理**所有其他类型的能力间关系**。

### 2.3 推荐表达形式

| 形式 | 适用场景 | 备注 |
|------|---------|------|
| **有向依赖图** | ⭐ 默认形式 | 节点=能力，边=依赖关系 |
| **矩阵表格** | 大量能力时更清晰 | 行=源能力，列=目标能力 |
| **分层分组图** | 强调逻辑分组时 | 先分组，再画组内/组间依赖 |

### 2.4 依赖类型区分

规范明确支持对依赖类型进行区分：

**图形化区分方法**：
- **颜色编码**连接线（不同颜色=不同类型）
- **虚线 vs 实线**（强依赖 vs 弱依赖）

**数据层面**：
- 优先使用 DM2 元模型中**预定义的依赖类型**
- 如预定义不足，可**新建专用依赖类型**
- 新类型必须在 **AV-2 集成词典**中注册记录

常见依赖类型示例：
| 类型 | 含义 | 示例 |
|------|------|------|
| 前置依赖 | A 必须在 B 之前就绪 | "态势感知" → "目标识别" |
| 协同依赖 | A 和 B 必须同时可用 | "通信" + "指挥控制" |
| 数据依赖 | A 的输出是 B 的输入 | "侦察" → "情报分析" |
| 约束依赖 | A 限制 B 的实现方式 | "安全策略" → "所有系统能力" |

### 2.5 完整细节

#### 用途场景

| 场景 | 说明 |
|------|------|
| **识别能力依赖** | 发现隐藏的依赖链路 |
| **能力管理** | 变更影响分析、处置决策等 |
| **采办项目交互** | 分析项目间依赖以达成整体能力 |

#### 架构图构建建议

1. 列出架构描述关注的所有能力
2. 基于**集成需求**将能力归入逻辑分组
3. 在组内和组间标注依赖关系
4. 用颜色或线型区分依赖类型
5. 保持图的清晰度——不要过度密集

```
    ↓ 时间+依赖

CV-6(活动映射) ← 怎么做？
CV-7(服务映射) ← 服务支撑？
```
