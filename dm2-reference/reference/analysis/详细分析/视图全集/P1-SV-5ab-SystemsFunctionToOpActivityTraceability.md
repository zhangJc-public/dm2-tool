# SV-5: 作战活动↔系统追溯矩阵（合卷）

## 概览

SV-5 是系统视点的**核心追溯桥梁**，由两个子视图组成：

```
┌──────────────────────────────────────────────────┐
│                 SV-5 追溯桥梁                      │
├──────────────────┬───────────────────────────────┤
│     SV-5a        │          SV-5b                 │
│ 活动→系统函数     │      活动→系统（含能力/执行者）  │
├──────────────────┼───────────────────────────────┤
│ 函数粒度          │ 系统/资源粒度                  │
│ 技术实现细节      │ 管理视角摘要                    │
│ 需求追溯          │ 采办/能力追溯                   │
└──────────────────┴───────────────────────────────┘
```

---

# SV-5a: Operational Activity to Systems Function Traceability Matrix

## 一、定位与目的

### 1.1 回答什么问题？

**一句话**：每个作战活动由哪些系统函数来支撑？函数级的精确映射。

SV-5a 是**最细粒度的活动-系统追溯**——从作战活动追溯到具体的系统功能函数。

### 1.2 核心特征

| 维度 | 说明 |
|------|------|
| **左轴** | Operational Activities（来自 OV-5a/b） |
| **右轴** | System Functions（来自 SV-4） |
| **单元格** | X / 日期 / 阶段（表示支撑关系） |

### 1.3 状态指示器 ⭐

规范定义了一套**交通灯符号**来表示实现状态：

| 颜色 | 含义 |
|------|------|
| 🔴 **Red** | 功能已规划但尚未开发 |
| 🟡 **Yellow** | 部分功能已提供（或完整功能已开发但未部署） |
| 🟢 **Green** | 完整功能已部署到现场 |
| ⬜ **Blank** | 无系统支撑计划 / 不存在关联 |

> ⚠️ 发布带状态信息的 SV-5a 时应**注明发布日期**，以便用户了解状态信息的时效性。

### 1.4 扩展标注

SV-5a 可进一步用以下信息标注：
- **执行活动的系统**
- **能力**（Capability）
- **执行者（Performer）**

# SV-5b: Operational Activity to Systems Traceability Matrix

## 一、定位与目的

### 1.1 回答什么问题？

**一句话**：每个作战活动由哪个系统来实现？系统级的追溯（比 SV-5a 更宏观）。

> SV-5b 描述 **SV-1 中的系统功能** 与 **OV-5a/b 中的作战活动** 之间的链接关系。
>
> SV-5b 展示了系统（以及可选的能力和执行者）向作战活动的**映射**。

### 1.2 核心价值 ⭐⭐

规范赋予 SV-5b 一个特殊的重要角色：

> 在**需求定义期间**，SV-5b 扮演特别重要的角色——**追溯与系统需求相关的架构元素** 到 **与用户需求相关的架构元素**。

换句话说：
- **OV 系列** = 用户需求侧（"我需要做什么"）
- **SV 系列** = 系统需求侧（"系统怎么实现"）
- **SV-5b** = 两者的**追溯桥梁**

### 1.3 多对多关系

> 作战活动和系统之间的关系预期为**多对多**：
> - 一个活动可由多个系统支撑
> - 一个系统可支持多个活动

### 1.4 SV-5a vs SV-5b 区别

| 维度 | SV-5a | SV-5b |
|------|-------|-------|
| **右侧元素** | System Functions（函数） | Systems（系统整体） |
| **粒度** | 更细（函数级） | 较粗（系统级） |
| **主要用途** | 技术实现追溯 | 管理/采办追溯 |
| **扩展信息** | 可加系统/能力/执行者 | 可隐藏系统只显示执行者和能力 |
| **关系** | SV-5b 可以是 SV-5a 的**摘要** | |

### 1.5 状态变体

与 SV-5a 类似，SV-5b 也支持交通灯状态：

| 颜色 | 含义 |
|------|------|
| 🔴 Red | 系统已规划但未开发 |
| 🟡 Yellow | 部分功能已提供（或已开发但未部署） |
| 🟢 Green | 完整功能已部署 |
| ⬜ Blank | 无系统支撑 / 不存在关联 |

### 1.6 能力追溯增强

> SV-5b 可进一步标注 Capabilities、Performers executing Activities、以及执行功能和能力的 Performers。
>
> 这可用于识别**哪些系统可以支撑特定能力**。

架构师甚至可以选择**隐藏系统**，让表格仅展示：
- 执行活动的 Performer
- 能力和 Performer → Operational Activities

## 二、SV-5 在完整追溯链中的位置

```
CV-6 (能力→活动)
    ↓
OV-5b (活动模型)
    ↓ ═══ SV-5 追溯桥 ═══
SV-5a (活动→系统函数) ← 细粒度
SV-5b (活动→系统)    ← 粗粒度
    ↓
SV-4 (系统功能模型)
SV-1 (系统接口)
```

### 与 CV-6/SvcV-5 的镜像对称

```
能力侧:           作战侧:           系统/服务侧:
  CV-6 ──→ OV活动 ──→ SV-5a/SV-5b  (系统)
                                  SvcV-5       (服务)
  CV-7 ──→ (直接)  ──→            SvcV-5       (服务)
```

| 上游依赖 | 下游支撑 | 同级互补 |
