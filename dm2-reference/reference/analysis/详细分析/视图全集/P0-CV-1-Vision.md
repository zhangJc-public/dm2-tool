# CV-1: Vision（愿景）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：我们要**往哪里去**？为什么需要变革？

CV-1 是能力视点的**战略顶层**——定义一组能力的**总体战略上下文**和**变革蓝图**。它比 OV-1 的场景式范围更通用、更抽象。

核心问题集：
| 问题 | 回答 |
|------|------|
| 我们想达成什么？ | 愿景（Vision） |
| 战略目标是什么？ | 目标（Goals） |
| 具体要实现什么？ | 目的（Objectives） |
| 如何衡量成功？ | 期望结果（Outcomes）、可度量收益（Measurable Benefits） |
| 时间范围是什么？ | 有界时间段 |

### 1.2 在架构描述中的角色

```
┌────────────────────────────────────────────┐
│          CV-1: 愿景                        │
│     （能力视点的战略锚点 / 变革蓝图）        │
├────────────────────────────────────────────┤
│                                            │
│   愿景 → 目标 → 目的 → 计划 → 活动         │
│    ↓      ↓       ↓       ↓      ↓        │
│   战略方向  期望成果  可度量收益  行动项     │
│                                            │
│   ┌──────────────────────────────┐        │
│   │ CV-2 能力分类法              │ ← 从属 │
│   │ CV-3 能力阶段               │ ← 时间 │
│   │ CV-4~7 映射视图            │ ← 展开 │
│   └──────────────────────────────┘        │
│                                            │
└────────────────────────────────────────────┘
```

**关键角色**：
- **能力视点的入口和边界定义者**
- **变革计划的顶层蓝图**
- **所有其他 CV 视图的"北极星"**
- **比 OV-1 更高层级的范围定义**（OV-1 = 场景级，CV-1 = 战略级）

## 二、核心内容解析

### 2.1 必要性说明（官方 Purpose 翻译）

> CV-1 处理与企业**转型努力相关的整体愿景**的企业关注点，从而为一组能力定义**战略上下文**。
>
> CV-1 的目的是为 Capability Viewpoint 架构描述中描述的能力提供**战略上下文**。它还为架构描述提供一个**比 OV-1 中定义的场景式范围更通用的高层范围**。
>
> **预期用途**：沟通关于能力开发的**战略愿景**。

### 2.2 推荐表达形式

| 形式 | 适用场景 | 备注 |
|------|---------|------|
| **文本描述** | ⭐ 默认形式 | 主要以文字形式呈现 |
| 愿景陈述文档 | 正式交付物 | 可能包含目标层级结构图 |
| 战略路线图 | 辅助图形 | 与 CV-3 联合使用时 |

### 2.3 关键数据元素（DM2 映射）

根据官方 Model Description，CV-1 描述以下概念：

| DM2 概念类别 | 在 CV-1 中的角色 |
|-------------|-----------------|
| **Vision（愿景）** | 核心内容——转型努力的整体愿景 |
| **Goals（目标）** | 高层战略方向 |
| **Objectives（目的）** | 可衡量的具体目标 |
| **Plans（计划）** | 实现路径 |
| **Activities（活动）** | 关键行动项 |
| **Events（事件）** | 关键里程碑/触发条件 |
| **Conditions（条件）** | 前提假设 |
| **Measures（度量）** | 衡量标准 |
| **Effects/Outcomes（效果/产出）** | 期望的结果 |
| **Produced Objects（产出物）** | 变革将产生的实物/服务 |

### 2.4 完整细节

#### 官方详细描述翻译：

> CV-1 通过概述**有界时间段内某能力领域的愿景**来定义架构描述中描述的一组能力的**战略上下文**。它描述高层目标和战略如何以**能力术语**交付。
>
> CV-1 可以作为**转型倡议（Transformational Initiative）的蓝图**。CV-1 可能主要是企业所参与的转型或变更计划的**总体目标的文本描述**。
>
> **关键重要性**：识别** Goals（目标）** 以及与之关联的 **Desired Outcomes（期望结果）** 和 **Measurable Benefits（可度量收益）**。

#### CV-1 的本质特征：
1. **时间有界性**——不是永恒的愿景，而是在特定时期内的战略方向
2. **能力导向**——用"能力"而非"系统"或"项目"来表达
3. **变革导向**——通常与转型计划绑定
4. **文本为主**——不像 OV-1 以图形为核心，CV-1 以文字叙述为主

```

  ├──→ CV-2 (能力分类法) —— "我们要什么能力"

  ├──→ CV-4 (能力依赖)     —— "哪些能力互相依赖"
  ├──→ CV-5 (能力→组织)    —— "谁来做"
  ├──→ CV-6 (能力→活动)    —— "做什么活动支撑"
  └──→ CV-7 (能力→服务)    —— "什么服务使能"
```
