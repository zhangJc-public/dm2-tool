# SvcV-4: Services Functionality Description（服务功能描述）

## 一、定位与目的

### 1.1 这张图回答什么问题？

**一句话**：每个服务**做什么功能**？输入输出数据流是什么？

SvcV-4 是服务视点的**行为/功能模型**——与 SvcV-1（结构）互补，详细描述服务的功能分解和数据流动。

核心问题集：
| 问题 | 回答 |
|------|------|
| 每个服务执行什么功能？ | 功能层级分解 |
| 功能之间如何通过数据连接？ | 数据流图（DFD） |
| 所有必需的输入都满足了吗？ | 完整性检查 |
| 人机交互在哪里？ | HCI/GUI 功能 |

### 1.2 在架构描述中的角色

```
┌──────────────────────────────────────────────┐
│        SvcV-4: 服务功能描述                   │
│   （行为模型 / SvcV-1 的功能对应物）           │
├──────────────────────────────────────────────┤
│                                              │
│   与 OV-5b 的平行关系:                        │
│   OV-5b (作战活动) ──实现──→ SvcV-4 (服务功能) │
│   "What (业务)"              "How (服务)"     │
│                                              │
│   两种表达形式:                                │
│   ├─ 分类法功能层次树 (Taxonomic Hierarchy)    │
│   │   └─ 树形结构, 适合并发依赖任务(如生产线)  │
│   └─ 数据流图 (Data Flow Diagram)             │
│       └─ 功能 + 数据流箭头 + 数据存储          │
│                                              │
│   文档内容:                                   │
│   • 服务功能 (Service Functions)               │
│   • 功能间的 Resource Flows                    │
│   • 内部服务数据仓库                           │
│   • 外部数据源/汇点(架构范围内)                 │
│   • 用户与服务的关系行为                       │
│                                              │
└──────────────────────────────────────────────┘
```

**关键定位**：
- **SvcV-1 的行为对应物**（如同 OV-5b 对应 OV-2）
- **OV-5b 的服务层实现**（作战活动→服务功能的映射目标端）
- 支持以网络为中心的实现中的**服务注册**

## 二、核心内容解析

### 2.1 必要性说明

> SvcV-4 解决**人工和服务功能**问题。
>
> **主要目的**：
> 1. 开发清晰的**必要数据流描述**——每个资源的输入（消费）和输出（生产）
> 2. 确保**服务功能连通性完整**（即资源的所有必需输入都已满足）
> 3. 确保**功能分解达到适当的详细程度**

预期用途：
1. 任务工作流描述
2. 功能性服务需求识别
3. 服务功能分解
4. 关联人工和服务功能

### 2.2 推荐表达形式

| 形式 | 适用场景 | 说明 |
|------|---------|------|
| **分类法功能层次树** | 并发但依赖的任务 | 如生产线的任务分解 |
| **数据流图（DFD）** | ⭐ 通用默认 | 功能 + 数据流箭头 + 数据存储 |
| **带泳道的 DFD 变体** | 多资源场景 | 按系统/能力分组展示 |

### 2.3 关键数据元素

| 元素 | 来源映射 |
|------|---------|
| Service Functions（服务功能） | 来自 SvcV-4 自身定义；可叠加到 SvcV-1 |
| Resource Flows between Functions | → SvcV-6 详细记录 |
| Internal Service Data Stores/Repositories | → DIV-3 物理来源 |
| External Sources/Sinks（架构范围内） | 外部交互边界 |

### 2.4 完整细节

#### 范围选项

| 范围类型 | 说明 |
|---------|------|
| **能力范围** | 不考虑哪个资源执行哪个功能 |
| **资源特定** | 聚焦特定资源的功能 |
| **内部数据流焦点** | 资源内的数据流动 |
| **外部数据流焦点** | 跨资源的数据交换 |
| **仅功能分配** | 只将功能分配给资源 |

#### 与 OV-5b 的关系

> SvcV-4 是 **OV-5b Operational Activity Model 在作战视点的对应物在服务视点中的版本**。
>
> ⚠️ 虽然 Operational Activity Model (OV-5b) 和 SV-4/SvcV-4 的功能层次之间存在相关性，但它**不需要是一对一映射**——因此需要 Function to Operational Activity Traceability Matrix（SvcV-5/SV-5）来提供该映射。
