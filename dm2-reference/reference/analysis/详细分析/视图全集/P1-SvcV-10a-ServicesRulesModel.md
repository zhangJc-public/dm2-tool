# SvcV-10a: Services Rules Model（服务规则模型）

## 一、定位与目的

### 1.1 这个模型回答什么问题？

**一句话**：服务行为受哪些技术规则约束？（SLA 策略、访问控制、编排约束等）

SvcV-10a 是服务视点的**规则库**——捕获约束服务行为的技术性规则。

### 1.2 与 OV-6a / SV-10a 的三层分工 ⭐⭐

| 维度 | **OV-6a** | **SV-10a** | **SvcV-10a** |
|------|-----------|------------|--------------|
| **What / How** | **What** (使命约束) | **How** (系统实施) | **How** (服务契约) |
| **抽象层级** | 条令/业务级 | 技术/实现级 | **服务/编排级** |
| **示例** | "BDA 仅晴天执行" | "TLS 1.3 only" | "超时 30s 自动重试 3 次" |
| **典型规则** | 交战规则 | 接口安全策略 | SLA/重试/熔断/降级 |

### 1.3 服务规则的特色领域

SvcV-10a 除了通用技术规则外，还涵盖**服务特有的**规则类型：

| 规则类别 | 示例 |
|---------|------|
| **SLA 策略** | "响应时间 P99 < 200ms" |
| **服务编排约束** | "A 必须在 B 之后调用" |
| **熔断/降级** | "错误率 >5% 时触发熔断" |
| **重试策略** | "指数退避，最大 3 次" |
| **服务发现** | "必须通过服务注册中心路由" |
| **版本兼容** | "API 版本向后兼容 2 个大版本" |

## 二、核心内容解析

### 2.1 规则表达

与 OV-6a/SV-10a 类似的两种形式：
- **命令性**："服务调用**必须**通过 API Gateway"
- **条件命令性**：**If** 返回 503 **then** 重试并降级到备用端点

### 2.2 与 SvcV-3b 的关系

> SvcV-3b 支持 net-centric 实现作为 **SvcV-10a/b/c**（作为服务**编排**实现的输入）。

SvcV-10a 规则直接指导服务编排引擎的行为。

### 规则体系的完整六层

```
条令层:    OV-6a (作战规则)     = What: 使命约束
           ↓
系统实现层: SV-10a (系统规则)    = How: 系统遵守方式
服务契约层: SvcV-10a (服务规则)  = How: 服务遵守方式  ← 本视图
           ↓
运行时代码: 配置/策略/编排脚本
```
