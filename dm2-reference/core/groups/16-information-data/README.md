# Information And Data（信息与数据）

> 📄 **完整分析** → [[../详细分析/DM2-InformationAndData详细分析]]

DM2 的信息本体——**Peirce 符号学三元模型**的形式化。

## 三层表示链（最核心！）

```
Sign（符号/标记）                          ← 人可直接感知
    ↑ representedBy                        例："机密"红色印章、二维码
Representation（表示形式）                 ← 数据的编码方式
    ↑ representedBy                        例：PDF 文件、数据库行、JSON
Data（数据内容）                            ← 语义化的信息
                                            例："该系统三级等保"
```

## 每一层都可以独立存在

| 场景 | 有 Sign 无 Rep？ | 有 Rep 无 Data？ |
|------|------------------|------------------|
| 看到印章但文件损坏 | ✅ 可能 | — |
| 收到加密文件但无法解密 | — | ✅ 可能 |
| 正常情况 | ✅ | ✅ |

## vs 04-Resource 的边界

| 角度 | 04-Resource | 16-InfoData |
|------|------------|-------------|
| 问题 | 这资源谁用？在哪？多敏感？ | 这信息怎么表示？本质是啥？ |
| 侧重点 | 管理视角（访问控制） | 本体视角（语义结构） |
| 同一实体的两面 | ✅ 可以两边都有条目 |
