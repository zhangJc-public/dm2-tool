# 00-基础模式

## 📌 一句话说明
DM2 的"语法层"——IDEAS 本体公理、关联模式、命名规范、时间建模等所有数据组遵循的元模式。

## 🎯 目录用途
- 存储 DM2 的元元模型（Meta-Meta Model）层内容
- 为其他 16 个数据组提供形式化基础
- 作为"模式速查手册"供日常建模参考
- 不包含业务实例，只含模式定义和提炼

## 📂 结构一览
```
00-基础模式/
├── AGENT.md                          ← 本文件
├── README.md                         ← 总览导航页（5张模式图入口）
├── IDEAS-TopLevel.md                 ← IDEAS 四分类本体公理（Thing/Individual/Type/Tuple）
├── FoundationForAssociations.md      ← 关联语法（5种核心关联模式的定义）
├── CommonPatterns.md                 ← Instance/Type 双面板模式对照
├── NamingAndDescriptionPattern.md    ← 命名、描述、表示、方案四区域模式
└── TemporalPartAndBoundaries.md      ← 时间组成与边界建模
```

## 🔗 关联目录
| 关联 | 目录 | 关系 |
|------|------|------|
| 被依赖 | `01-Performer` ~ `16-InfoData` | 所有数据组的建模都遵循本层的模式规则 |
| 深度分析 | `../详细分析/` | 每个模式都有对应的详细分析文档 |

## 🤖 Agent 协作规则
1. **本目录只读优先**：修改前先确认是否与 `详细分析/` 中的深度文档一致
2. **提炼原则**：本目录的内容应该是详细分析的**精华摘要**，不是简单复制
3. **不创建实例**：本目录不放具体实体笔记，只放模式和规范
4. **跨目录引用**：当其他目录需要引用"某个模式怎么用"时，链接到这里
