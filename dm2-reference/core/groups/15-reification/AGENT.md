# 15-ReificationLevels

## 📌 一句话说明
**具象化——层级转换规则。** Type ↔ Individual 之间如何"变身"的形式化规则集。

## 🎯 目录用途
- 定义 DM2 的四层 Reification 体系
- 解释 TypeType → Type → Individual 之间的转换约束
- 解决"这个实体应该在哪一层"的常见困惑

## 四层速查

| 层级 | 含义 | 示例 | 典型用户 |
|------|------|------|---------|
| **TypeType** | 类型之类型 | "能力类型"的概念本身 | 架构设计师 |
| **Type** | 类别/模板 | "网络防御能力" | 系统分析师 |
| **Individual** | 具体实例 | "某SOC的入侵检测能力" | 项目经理 |
| **IndividualIndividual** | 实例的特定出现 | "2026Q1的入侵检测运行记录" | 运维人员 |
