# 设计方

## 角色定义

**设计方**是负责系统架构设计、安全方案编制的咨询或设计单位。

## DM2 类型层级

- **Organization-Type**：`设计方`
- **关联**：performerTypeOfIndividual → Organization-Individual（具体设计单位）

## 职责范围

| 职责 | 说明 |
|------|------|
| 需求分析 | 调研业务需求、合规要求 |
| 架构设计 | 设计安全体系架构 |
| 方案编制 | 编写技术方案、设计文档 |
| 评审支持 | 参与方案评审与优化 |

## 典型关切点

- ✅ 需求是否完整、清晰
- ✅ 设计方案是否可行
- ✅ 技术选型是否合理
- ✅ 是否符合行业标准

## 关联角色

| 角色 | 关系 |
|------|------|
| 甲方 | 服务对象 |
| 乙方 | 协同设计 |
| 建设方 | 设计交底 |
| 监管方 | 符合监管要求 |
